BESPA

Backplane for Electromicroscopy Single Particle Analysis

BESPA is a software package for image and volume data processing in single particle analysis.

Overview

Single particle analysis is a method for structural studies of protein and biological macromolecules. From observed images of the isolated molecules, the three dimensional volume can be reconstructed. The method was well demonstrated by pioneering works on ribosome, the molecular machine made of RNA and protein complex. In order to support computational analysis numbers of software systems available in both free distributions and commercial package.

Currently, BESPA is a set of programs for image processing and three dimensional reconstruction, and they are controled by a script interpreter. There are still development of programs, refinementm and packaging project underway. We think that our sotware would contiribute some of scientific activities, if the development continues through discussions and collaboration although the release code is prerilimary. Angular Reconstitution

From projection images in random orientation of the same molecules, the angular reconstituion method tries to reconstruct volume consistent to observed images. Following to the leteratures, this implementation is to automate this task using around 20 projection images. So far, it is ok for an ideal case with fairly good signal to noise ratio.

Separating Projection Images in the Different Conformations

Projection images from the same object at the same conformation are required for this statistical reconstruction technique. However, some molecules exhibit different conformations that probably reflect their activity. A method is proposed to separate single particle images in the different conformations. Recently, we have studied method using common line matchng score to separarte particle images in the different conformations.


Scripting Language

In order to support various tasks of image processing and volume reconstrucitons, a scrpit interpreter is provided. Lua language was employed because its extensibility nicely fits to our original software development.

Visualization of Volume and Images

To support comparisons and examination of image processing, variations of density maps, volume rendering and isosurface display for multiple data are supported. Preliminary GUI controls are also available on Linux, Windows and MacOS(X11) platforms.

Parallel Processing

Image and volume data are ideal targets of the parallel computing methods. Our code also offer parallel computing based on our original parallel computing library. CPEL, Comprehensive Parallel Excution Library.
